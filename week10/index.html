CLIENT-SIDE VALIDATION

Client-side form validation help ensure data submitted matches the requirements set forth in the various form controls. Users can fix client-side validation right away if something is not input correctly. Validation done on the server is called server-side validation. Data passed from the client to the server should never be trusted.

Built-in Validation Attributes

Required, min/maxlength, min & max, type, pattern (regular expression). CSS pseudo class :valid applies specific style to valid elements. :invalid also can helps users see if something is not valid.

Basic examples of how regular expression works:

  • a — Matches one character that is a (not b, not aa, and so on).
  • abc — Matches a, followed by b, followed by c.
  • ab?c—Matches a, optionally followed by a single b, followed by c. ( ac or abc)
  • ab*c—Matches a, optionally followed by any number of bs, followed by c. ( ac , abc, abbbbbc, and so on).
  • a|b — Matches one character that is a or b.
  • abc|xyz — Matches exactly abc or exactly xyz (but not abcxyz or a or y, and so on).

To validate a form, ask yourself a few questions:

  • What kind of validation should I perform?
  • What should I do if the form doesn't validate?
  • How can I help the user to correct invalid data?

In order to reduce the user's frustration, it's very important to provide as much helpful information as possible in order to guide them in correcting their inputs. You should offer up-front suggestions so they know what's expected, as well as clear error messages.

USING THE FETCH API

The Fetch API provides a JavaScript interface for accessing and manipulating parts of the HTTP pipeline, such as requests and responses. The fetch() method can optionally accept a second parameter, an init object that allows youn to control a number of different settings. Files can be uploaded using an HTML input type="file" multiple input element, FormData() and fetch().
